# Activity 도메인 트러블슈팅 시나리오

## 발생 가능한 문제 상황들

### 1. 활동 로그 대량 저장 시 성능 저하
- **시나리오**: 활동 로그를 개별 INSERT로 저장하여 성능 저하
- **재현 방법**: 많은 활동 로그 저장 시 응답 시간 측정
- **예상 증상**: 응답 시간 증가, DB 부하 증가

### 2. 활동 로그 테이블 크기 증가
- **시나리오**: 활동 로그가 계속 쌓일 때 테이블 크기 증가
- **재현 방법**: 오래된 활동 로그 확인
- **예상 증상**: 테이블 크기 증가, 조회 성능 저하

### 3. 활동 로그 중복 기록
- **시나리오**: 같은 활동을 짧은 시간 내 여러 번 수행할 때 중복 기록
- **재현 방법**: 같은 활동을 짧은 시간 내 여러 번 수행
- **예상 증상**: 통계 왜곡

### 4. 활동 로그 조회 시 페이징 처리 누락
- **시나리오**: 활동 로그 조회 시 모든 로그를 한 번에 조회
- **재현 방법**: 활동 로그가 많을 때 조회
- **예상 증상**: 응답 시간 증가, 메모리 사용량 증가

### 5. 활동 로그 통계 조회 성능 저하
- **시나리오**: 활동 로그 통계 조회 시 매번 집계 쿼리 실행
- **재현 방법**: 활동 로그 통계 조회 API 호출 시 쿼리 확인
- **예상 증상**: 매번 집계 쿼리 발생, 응답 시간 증가

### 6. 활동 로그 비동기 처리 실패
- **시나리오**: 활동 로그 비동기 처리 중 예외 발생 시 로그 손실
- **재현 방법**: 비동기 처리 중 예외 발생
- **예상 증상**: 활동 로그 손실

### 7. 활동 로그 타입 검증 누락
- **시나리오**: 활동 로그 생성 시 유효한 활동 타입인지 확인하지 않음
- **재현 방법**: 잘못된 활동 타입으로 로그 생성 시도
- **예상 증상**: 잘못된 활동 로그 생성 가능

### 8. 활동 로그 삭제 시 통계 데이터 손실
- **시나리오**: 활동 로그 삭제 시 통계 데이터도 함께 삭제
- **재현 방법**: 활동 로그 삭제 후 통계 확인
- **예상 증상**: 통계 데이터 손실
