# Board 도메인 트러블슈팅 시나리오

## 발생 가능한 문제 상황들

### 1. 게시글 목록 조회 시 N+1 문제
- **시나리오**: 게시글 목록 조회 시 각 게시글의 좋아요/싫어요 수를 개별 쿼리로 조회
- **재현 방법**: 게시글 목록 API 호출 시 로그 확인
- **예상 증상**: 게시글 100개 조회 시 200개 이상의 쿼리 발생

### 2. 조회수 중복 증가
- **시나리오**: 같은 사용자가 게시글을 여러 번 조회할 때마다 조회수 증가
- **재현 방법**: 같은 사용자로 게시글을 여러 번 조회
- **예상 증상**: 새로고침할 때마다 조회수 증가

### 3. 인기글 조회 성능 저하
- **시나리오**: 인기글 조회 시 매번 복잡한 계산 수행
- **재현 방법**: 인기글 조회 API 호출 시 응답 시간 측정
- **예상 증상**: 응답 시간 3초 이상

### 4. 댓글 수 동기화 문제
- **시나리오**: 여러 사용자가 동시에 댓글 작성/삭제 시 commentCount 불일치
- **재현 방법**: 동시에 여러 댓글 작성/삭제
- **예상 증상**: 게시글의 commentCount와 실제 댓글 수 불일치

### 5. 좋아요 카운트 실시간 업데이트 동시성 문제
- **시나리오**: 여러 사용자가 동시에 좋아요 클릭 시 likeCount 부정확
- **재현 방법**: 동시에 여러 사용자가 좋아요 클릭
- **예상 증상**: likeCount가 부정확하게 업데이트됨

### 6. 게시글 검색 성능 저하
- **시나리오**: LIKE 쿼리 사용 시 인덱스 미사용, 전체 테이블 스캔
- **재현 방법**: 게시글 검색 API 호출 시 쿼리 실행 계획 확인
- **예상 증상**: 검색 응답 시간 5초 이상

### 7. 게시글 첨부파일 조회 N+1 문제
- **시나리오**: 게시글 목록 조회 시 각 게시글의 첨부파일을 개별 쿼리로 조회
- **재현 방법**: 게시글 목록 API 호출 시 로그 확인
- **예상 증상**: 게시글마다 첨부파일 조회 쿼리 발생

### 8. 게시글 삭제 시 연관 데이터 처리 누락
- **시나리오**: 게시글 삭제 시 댓글, 좋아요 등 연관 데이터 미처리
- **재현 방법**: 게시글 삭제 후 연관 데이터 확인
- **예상 증상**: 삭제된 게시글의 댓글이나 좋아요가 남아있음
