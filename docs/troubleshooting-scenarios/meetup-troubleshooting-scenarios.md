# Meetup 도메인 트러블슈팅 시나리오

## 발생 가능한 문제 상황들

### 1. 최대 인원 초과
- **시나리오**: 여러 사용자가 동시에 모임 참여 신청 시 최대 인원 초과
- **재현 방법**: 동시에 여러 사용자가 참여 신청
- **예상 증상**: 최대 인원을 초과하여 참여 신청됨

### 2. 모임 목록 조회 N+1 문제
- **시나리오**: 모임 목록 조회 시 각 모임의 주최자, 참여자 수를 개별 쿼리로 조회
- **재현 방법**: 모임 목록 조회 시 로그 확인
- **예상 증상**: 모임마다 연관 데이터 조회 쿼리 발생

### 3. 위치 기반 검색 성능 저하
- **시나리오**: 위치 기반 모임 검색 시 모든 모임을 조회한 후 거리 계산하여 필터링
- **재현 방법**: 모임이 많을수록 검색 성능 저하
- **예상 증상**: 검색 응답 시간 증가

### 4. 모임 상태 업데이트 동시성 문제
- **시나리오**: 여러 사용자가 동시에 모임 상태 변경 시도
- **재현 방법**: 동시에 모임 상태 변경 API 호출
- **예상 증상**: 상태 업데이트 불일치

### 5. 모임 만료 처리 누락
- **시나리오**: 모임 만료 시 자동 완료 처리 누락
- **재현 방법**: 만료된 모임 확인
- **예상 증상**: 만료된 모임이 계속 RECRUITING 상태로 남아있음

### 6. 모임 참여 취소 시 인원 수 처리 문제
- **시나리오**: 모임 참여 취소 시 currentParticipants 감소 처리 누락
- **재현 방법**: 모임 참여 취소 후 인원 수 확인
- **예상 증상**: currentParticipants가 감소하지 않음

### 7. 모임 생성 시 날짜 검증 누락
- **시나리오**: 모임 생성 시 과거 날짜로 생성 가능
- **재현 방법**: 과거 날짜로 모임 생성 시도
- **예상 증상**: 과거 날짜로 모임 생성 가능

### 8. 모임 참여자 목록 조회 N+1 문제
- **시나리오**: 모임 참여자 목록 조회 시 각 참여자의 정보를 개별 쿼리로 조회
- **재현 방법**: 모임 참여자 목록 조회 시 로그 확인
- **예상 증상**: 참여자마다 정보 조회 쿼리 발생
