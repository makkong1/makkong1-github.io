# Payment 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **PetCoinService** | 충전 금액은 0보다 커야 합니다 | IllegalArgumentException | 400 |
| | User not found | RuntimeException | 404 |
| | 차감 금액은 0보다 커야 합니다 | IllegalArgumentException | 400 |
| | 잔액이 부족합니다. 현재 잔액: %d, 필요 금액: %d | IllegalStateException | 400 |
| | 지급 금액은 0보다 커야 합니다 | IllegalArgumentException | 400 |
| | 환불 금액은 0보다 커야 합니다 | IllegalArgumentException | 400 |
| | 거래 내역을 찾을 수 없습니다 | RuntimeException | 404 |
| | 본인의 거래 내역만 조회할 수 있습니다 | RuntimeException | 403 |
| **PetCoinEscrowService** | 에스크로 금액은 0보다 커야 합니다 | IllegalArgumentException | 400 |
| | 이미 에스크로가 생성되어 있습니다 | IllegalStateException | 409 |
| | Escrow not found | RuntimeException | 404 |
| | HOLD 상태의 에스크로만 지급할 수 있습니다 | IllegalStateException | 409 |
| | HOLD 상태의 에스크로만 환불할 수 있습니다 | IllegalStateException | 409 |
| **PetCoinController** | 충전 금액은 0보다 커야 합니다 | IllegalArgumentException | 400 |
| | 인증이 필요합니다 | RuntimeException | 401 |
| | User not found | RuntimeException | 404 |
| **AdminPaymentController** | 사용자 ID가 필요합니다 | IllegalArgumentException | 400 |
| | 충전 금액은 0보다 커야 합니다 | IllegalArgumentException | 400 |
| | User not found | RuntimeException | 404 |

### 1.2 문제점
1. **예외 타입 혼용**: RuntimeException vs IllegalArgumentException vs IllegalStateException
2. **메시지 혼재**: "Escrow not found" (영어) vs "거래 내역을 찾을 수 없습니다" (한글)
3. **User 도메인 의존**: UserNotFoundException, UnauthenticatedException 재사용 필요
4. **금액 검증 반복**: 충전/차감/지급/환불/에스크로 모두 "0보다 커야 합니다" 패턴

## 2. 도입할 예외 클래스

### 2.1 구조
```
domain/payment/exception/
├── PetCoinTransactionNotFoundException.java # 404 - 거래 내역 미존재
├── PetCoinEscrowNotFoundException.java      # 404 - 에스크로 미존재
├── PaymentForbiddenException.java           # 403 - 본인의 거래 내역만 조회 가능
├── PaymentValidationException.java          # 400 - 금액 검증, 사용자 ID 필수 등
├── PaymentConflictException.java            # 409 - 이미 에스크로 생성, HOLD 상태 아님 등
└── InsufficientBalanceException.java        # 400 - 잔액 부족
```

### 2.2 User 도메인 예외 재사용
- **UnauthenticatedException**: 인증 필요 (PetCoinController)
- **UserNotFoundException**: 사용자 조회 실패 시

### 2.3 ApiException 베이스
- `global/exception/ApiException.java` 상속 (기존 유지)

## 4. 예외 정적 메서드 제안

### PetCoinTransactionNotFoundException
```java
PetCoinTransactionNotFoundException // 거래 내역을 찾을 수 없습니다
```

### PetCoinEscrowNotFoundException
```java
PetCoinEscrowNotFoundException // 에스크로를 찾을 수 없습니다
```

### PaymentForbiddenException
```java
PaymentForbiddenException.ownTransactionOnly() // 본인의 거래 내역만 조회할 수 있습니다
```

### PaymentValidationException
```java
PaymentValidationException.userIdRequired()     // 사용자 ID가 필요합니다
PaymentValidationException.chargeAmountInvalid()  // 충전 금액은 0보다 커야 합니다
PaymentValidationException.deductAmountInvalid()   // 차감 금액은 0보다 커야 합니다
PaymentValidationException.payoutAmountInvalid()  // 지급 금액은 0보다 커야 합니다
PaymentValidationException.refundAmountInvalid()  // 환불 금액은 0보다 커야 합니다
PaymentValidationException.escrowAmountInvalid()  // 에스크로 금액은 0보다 커야 합니다
```

### PaymentConflictException
```java
PaymentConflictException.escrowAlreadyExists()  // 이미 에스크로가 생성되어 있습니다
PaymentConflictException.holdStatusRequiredForRelease()  // HOLD 상태의 에스크로만 지급할 수 있습니다
PaymentConflictException.holdStatusRequiredForRefund()    // HOLD 상태의 에스크로만 환불할 수 있습니다
```

### InsufficientBalanceException
```java
InsufficientBalanceException(int current, int required)  // 잔액이 부족합니다. 현재 잔액: %d, 필요 금액: %d
```
