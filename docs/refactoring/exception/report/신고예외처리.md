# Report 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **ReportService** | 신고 대상 종류를 선택해주세요 | IllegalArgumentException | 400 |
| | 신고 대상 ID가 필요합니다 | IllegalArgumentException | 400 |
| | 신고자 정보가 필요합니다 | IllegalArgumentException | 400 |
| | 신고 사유를 입력해주세요 | IllegalArgumentException | 400 |
| | 신고자 정보를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 이미 해당 대상을 신고하셨습니다 | IllegalStateException | 409 |
| | 신고 정보를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 처리 상태를 선택해주세요 | IllegalArgumentException | 400 |
| | 관리자 정보를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 신고 대상 게시글을 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 신고 대상 댓글을 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 신고 대상 실종 제보를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 해당 서비스 제공자를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 서비스 제공자만 신고할 수 있습니다 | IllegalArgumentException | 400 |
| | 지원하지 않는 신고 대상입니다 | IllegalArgumentException | 400 |
| **ReportAssistAgentService** | catch (Exception) - LLM 호출 | - | Optional.empty() |
| | catch (IllegalArgumentException) - parseActionType | - | ReportActionType.NONE |

### 1.2 문제점
1. **예외 타입 혼용**: IllegalArgumentException vs IllegalStateException
2. **User 도메인 의존**: UserNotFoundException (신고자, 관리자) 재사용 필요
3. **다른 도메인 의존**: Board, Comment, MissingPetBoard, Care 등 대상 검증 시 해당 도메인 NotFound 예외 또는 Report 전용
4. **AI 에이전트**: ReportAssistAgentService는 LLM 실패 시 Optional.empty() 반환 (예외 전파 안 함)

## 2. 도입할 예외 클래스

### 2.1 구조
```
domain/report/exception/
├── ReportNotFoundException.java      # 404 - 신고 정보 미존재
├── ReportValidationException.java     # 400 - 대상 종류/ID/사유/처리 상태 필수, 지원하지 않는 대상 등
├── ReportTargetNotFoundException.java # 404 - 신고 대상(게시글/댓글/실종제보/제공자) 미존재
├── ReportForbiddenException.java      # 400 - 서비스 제공자만 신고 가능 (역할 검증)
└── ReportConflictException.java      # 409 - 이미 해당 대상 신고함
```

### 2.2 User 도메인 예외 재사용
- **UserNotFoundException**: 신고자, 관리자, 서비스 제공자 조회 실패 시

### 2.3 Board/Care 도메인 예외 재사용 (선택)
- BoardNotFoundException, CommentNotFoundException, MissingPetBoardNotFoundException 등
- 또는 ReportTargetNotFoundException으로 통합 (메시지만 구분)

### 2.4 ApiException 베이스
- `global/exception/ApiException.java` 상속 (기존 유지)

## 4. 예외 정적 메서드 제안

### ReportNotFoundException
```java
ReportNotFoundException // 신고 정보를 찾을 수 없습니다
```

### ReportValidationException
```java
ReportValidationException.targetTypeRequired()   // 신고 대상 종류를 선택해주세요
ReportValidationException.targetIdxRequired()    // 신고 대상 ID가 필요합니다
ReportValidationException.reporterRequired()      // 신고자 정보가 필요합니다
ReportValidationException.reasonRequired()         // 신고 사유를 입력해주세요
ReportValidationException.statusRequired()        // 처리 상태를 선택해주세요
ReportValidationException.unsupportedTarget()     // 지원하지 않는 신고 대상입니다
```

### ReportTargetNotFoundException
```java
ReportTargetNotFoundException.board()      // 신고 대상 게시글을 찾을 수 없습니다
ReportTargetNotFoundException.comment()    // 신고 대상 댓글을 찾을 수 없습니다
ReportTargetNotFoundException.missingPet() // 신고 대상 실종 제보를 찾을 수 없습니다
ReportTargetNotFoundException.provider()   // 해당 서비스 제공자를 찾을 수 없습니다
```

### ReportForbiddenException
```java
ReportForbiddenException.providerOnly() // 서비스 제공자만 신고할 수 있습니다
```

### ReportConflictException
```java
ReportConflictException.alreadyReported() // 이미 해당 대상을 신고하셨습니다
```
