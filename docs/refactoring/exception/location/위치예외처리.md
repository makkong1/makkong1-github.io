# Location 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **LocationServiceService** | 서비스를 찾을 수 없습니다 | RuntimeException | 404 |
| | 이미 삭제된 서비스입니다 | RuntimeException | 409 |
| **LocationServiceReviewService** | 이미 해당 서비스에 리뷰를 작성하셨습니다 | RuntimeException | 409 |
| | 서비스를 찾을 수 없습니다 | RuntimeException | 404 |
| | 사용자를 찾을 수 없습니다 | RuntimeException | 404 |
| | 이메일 인증이 필요합니다 | EmailVerificationRequiredException (user) | 403 |
| | 리뷰를 찾을 수 없습니다 | RuntimeException | 404 |
| | 이미 삭제된 리뷰입니다 | RuntimeException | 409 |
| **LocationServiceController** | catch (IllegalArgumentException) | - | 400 |
| | catch (Exception) - 검색 실패 | - | 500 |
| | catch (Exception) - AI 추천 실패 | - | 500 |
| | catch (Exception) - 삭제 실패 | - | 400 |
| **LocationServiceReviewController** | catch (Exception) - 전체 | - | 400 |
| **PublicDataLocationService** | CSV 파일 읽기 실패 | RuntimeException | 500 |
| **NaverMapService** | catch (HttpClientErrorException) | - | 외부 API |
| | catch (Exception) | - | 외부 API |
| **GeocodingController** | catch (UnsupportedEncodingException) | - | 400 |
| | catch (Exception) | - | 500 |
| **LocationRecommendAgentService** | catch (CompletionException, TimeoutException) | - | AI 에이전트 |
| | catch (Exception) | - | AI 에이전트 |
| **OperatingHoursParser** | catch (Exception) | - | 파싱 실패 |

### 1.2 문제점
1. **Controller try-catch 다수**: LocationServiceReviewController, LocationServiceController에서 모든 예외를 catch 후 수동 응답
2. **예외 타입 단일화**: 대부분 RuntimeException
3. **User 도메인 의존**: UserNotFoundException, EmailVerificationRequiredException 재사용 필요
4. **외부 연동 예외**: PublicDataLocationService(CSV), NaverMapService(API), GeocodingController 등 → 별도 처리 검토

## 2. 도입할 예외 클래스

### 2.1 구조
```
domain/location/exception/
├── LocationServiceNotFoundException.java   # 404 - 위치 서비스 미존재
├── LocationServiceReviewNotFoundException.java # 404 - 리뷰 미존재
├── LocationServiceAlreadyDeletedException.java  # 409 - 이미 삭제된 서비스
├── LocationReviewAlreadyDeletedException.java   # 409 - 이미 삭제된 리뷰
├── LocationReviewDuplicateException.java        # 409 - 이미 해당 서비스에 리뷰 작성
└── LocationValidationException.java             # 400 - 검색 파라미터 등 (선택)
```

### 2.2 User 도메인 예외 재사용
- **UserNotFoundException**: 사용자 조회 실패 시
- **EmailVerificationRequiredException**: 리뷰 작성/수정/삭제 시 이메일 인증 필요

### 2.3 ApiException 베이스
- `global/exception/ApiException.java` 상속 (기존 유지)

### 2.4 외부 연동 예외 (선택, 낮은 우선순위)
- PublicDataLocationService, NaverMapService, GeocodingController, LocationRecommendAgentService
- 외부 API/파일/AI 에러는 기존 catch 유지 또는 `LocationExternalServiceException` 등으로 래핑
