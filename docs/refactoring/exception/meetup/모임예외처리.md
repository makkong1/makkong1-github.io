# Meetup 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **MeetupController** | 인증된 사용자 정보를 찾을 수 없습니다 | RuntimeException | 401 |
| | catch (Exception) - 전체 API | - | 400 |
| **MeetupService** | 사용자를 찾을 수 없습니다 | RuntimeException | 404 |
| | 이메일 인증이 필요합니다 | EmailVerificationRequiredException (user) | 403 |
| | 모임 일시는 현재 시간 이후여야 합니다 | RuntimeException | 400 |
| | 모임을 찾을 수 없습니다 | RuntimeException | 404 |
| | 이미 참가한 모임입니다 | RuntimeException | 409 |
| | 모임 인원이 가득 찼습니다 | RuntimeException | 409 |
| | 주최자는 참가 취소할 수 없습니다 | RuntimeException | 403 |
| | 참가 정보를 찾을 수 없습니다 | RuntimeException | 404 |
| **MeetupConverter** | catch (IllegalArgumentException) | - | 변환 실패 |
| **MeetupChatRoomEventListener** | catch (Exception) | - | 이벤트 리스너 |

### 1.2 문제점
1. **Controller try-catch 다수**: MeetupController에서 모든 API에 catch (Exception) 후 수동 응답
2. **예외 타입 단일화**: 대부분 RuntimeException
3. **User 도메인 의존**: UserNotFoundException, EmailVerificationRequiredException 재사용 필요
4. **인증 예외**: "인증된 사용자 정보를 찾을 수 없습니다" → UnauthenticatedException (user)

## 2. 도입할 예외 클래스

### 2.1 구조
```
domain/meetup/exception/
├── MeetupNotFoundException.java      # 404 - 모임 미존재
├── MeetupParticipantNotFoundException.java # 404 - 참가 정보 미존재
├── MeetupValidationException.java    # 400 - 모임 일시 검증 등
├── MeetupForbiddenException.java     # 403 - 주최자는 참가 취소 불가
└── MeetupConflictException.java      # 409 - 이미 참가, 인원 가득 참
```

### 2.2 User 도메인 예외 재사용
- **UnauthenticatedException**: 인증된 사용자 정보 없음 (MeetupController)
- **UserNotFoundException**: 사용자 조회 실패 시
- **EmailVerificationRequiredException**: 모임 생성/참가 시 이메일 인증 필요

### 2.3 ApiException 베이스
- `global/exception/ApiException.java` 상속 (기존 유지)

## 4. 예외 정적 메서드 제안

### MeetupNotFoundException
```java
MeetupNotFoundException // 모임을 찾을 수 없습니다
```

### MeetupParticipantNotFoundException
```java
MeetupParticipantNotFoundException // 참가 정보를 찾을 수 없습니다
```

### MeetupValidationException
```java
MeetupValidationException.dateMustBeFuture() // 모임 일시는 현재 시간 이후여야 합니다
```

### MeetupForbiddenException
```java
MeetupForbiddenException.organizerCannotCancel() // 주최자는 참가 취소할 수 없습니다
```

### MeetupConflictException
```java
MeetupConflictException.alreadyJoined()         // 이미 참가한 모임입니다 (errorCode: MEETUP_ALREADY_JOINED)
MeetupConflictException.fullCapacity()           // 모임 인원이 가득 찼습니다 (errorCode: MEETUP_FULL)
MeetupConflictException.meetupNotRecruiting()    // 모집이 마감된 모임입니다 (errorCode: MEETUP_NOT_RECRUITING)
```

> 각 케이스 errorCode가 분리되어 프론트엔드에서 상황별 메시지 처리 가능.

### MeetupValidationException
```java
MeetupValidationException.dateMustBeFuture()         // 모임 일시는 현재 시간 이후여야 합니다 (생성·수정 공통)
MeetupValidationException.invalidMaxParticipants()   // 최대 참여 인원은 1 이상이어야 합니다
MeetupValidationException.maxBelowCurrent()          // 최대 인원은 현재 참여자 수보다 작을 수 없습니다
```
