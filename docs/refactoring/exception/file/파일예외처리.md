# File 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **FileStorageService** | 업로드 디렉터리를 초기화할 수 없습니다 | IllegalStateException | 500 |
| | File must not be empty | IllegalArgumentException | 400 |
| | 업로드 디렉터리를 준비하지 못했습니다 | RuntimeException | 500 |
| | 파일 저장에 실패했습니다. 잠시 후 다시 시도해주세요 | RuntimeException | 500 |
| | 요청한 파일을 찾을 수 없습니다: {path} | IllegalArgumentException | 404 |
| | 파일 경로가 비어 있습니다 | IllegalArgumentException | 400 |
| | 잘못된 파일 경로 요청입니다 | IllegalArgumentException | 400 |
| | 파일은 최대 5MB까지 업로드할 수 있습니다 | IllegalArgumentException | 400 |
| | 이미지 파일(jpg, png, gif, webp)만 업로드할 수 있습니다 | IllegalArgumentException | 400 |
| | 허용되지 않은 이미지 확장자입니다 | IllegalArgumentException | 400 |
| **AttachmentFileService** | catch (IllegalArgumentException ignored) | - | 내부 처리 |
| | catch (IOException \| IllegalArgumentException ignored) | - | 내부 처리 |
| **FileUploadController** | catch (IllegalArgumentException ex) → 404 | - | - |

### 1.2 문제점

1. **예외 타입 혼용**: IllegalStateException, RuntimeException, IllegalArgumentException 혼재
2. **메시지 혼재**: "File must not be empty" (영어) vs 한글 메시지
3. **IO 예외 래핑**: IOException → RuntimeException으로 래핑 (적절하나 ApiException 계층으로 통일 가능)
4. **Controller try-catch**: FileUploadController에서 IllegalArgumentException을 잡아 404 반환 → 예외 전파 후 GlobalExceptionHandler에서 처리 권장

## 2. 도입할 예외 클래스

### 2.1 구조

```
domain/file/exception/
├── FileNotFoundException.java          # 404 - 요청한 파일을 찾을 수 없음
├── FileStorageException.java           # 500 - 디렉터리 초기화/준비/저장 실패 (IOException 래핑)
├── FileValidationException.java       # 400 - 빈 파일, 경로 비어있음, 잘못된 경로
└── FileUploadValidationException.java # 400 - 용량 초과, 허용되지 않은 타입/확장자
```

### 2.2 ApiException 베이스

- `global/exception/ApiException.java` 상속 (기존 유지)

### 2.3 FileStorageException vs FileUploadValidationException

- **FileStorageException**: 서버 측 IO/시스템 오류 (500)
- **FileUploadValidationException**: 클라이언트 입력 검증 (용량, 타입, 확장자) (400)
- **FileValidationException**: 경로 검증, 빈 파일 등 일반 검증 (400)
- **FileNotFoundException**: 파일 미존재 (404)

## 4. 정적 메서드 제안

### FileValidationException

```java
FileValidationException.emptyFile()           // File must not be empty → 파일을 선택해주세요
FileValidationException.emptyPath()           // 파일 경로가 비어 있습니다
FileValidationException.invalidPath(String)   // 잘못된 파일 경로 요청입니다
```

### FileUploadValidationException

```java
FileUploadValidationException.sizeExceeded()  // 파일은 최대 5MB까지 업로드할 수 있습니다
FileUploadValidationException.invalidContentType() // 이미지 파일(jpg, png, gif, webp)만 업로드할 수 있습니다
FileUploadValidationException.invalidExtension()  // 허용되지 않은 이미지 확장자입니다
```

### FileStorageException

```java
FileStorageException.initFailed(Throwable)   // 업로드 디렉터리를 초기화할 수 없습니다
FileStorageException.prepareFailed(Throwable) // 업로드 디렉터리를 준비하지 못했습니다
FileStorageException.saveFailed(Throwable)    // 파일 저장에 실패했습니다. 잠시 후 다시 시도해주세요
```
