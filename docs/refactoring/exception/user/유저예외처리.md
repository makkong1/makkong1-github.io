# User 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **AuthService** | 유저 없음 | RuntimeException | 404 |
| | 영구 차단된 계정입니다 | RuntimeException | 403 |
| | 이용제한 중인 계정입니다. 해제일: %s | RuntimeException | 403 |
| | 유효하지 않은 Refresh Token | RuntimeException | 401 |
| | Refresh Token을 찾을 수 없습니다 | RuntimeException | 401 |
| | Refresh Token이 만료되었습니다 | RuntimeException | 401 |
| **UsersService** | User not found with username/id | RuntimeException | 404 |
| | 닉네임은 필수입니다 | RuntimeException | 400 |
| | 닉네임은 50자 이하여야 합니다 | RuntimeException | 400 |
| | 이미 사용 중인 닉네임/사용자명/이메일/아이디 | RuntimeException | 409 |
| | 비밀번호는 필수입니다 | RuntimeException | 400 |
| | 본인의 프로필만 수정할 수 있습니다 | RuntimeException | 403 |
| | 현재 비밀번호가 일치하지 않습니다 | RuntimeException | 400 |
| | 닉네임을 입력해주세요 | IllegalArgumentException | 400 |
| **EmailVerificationService** | 사용자를 찾을 수 없습니다 | RuntimeException | 404 |
| | 이메일이 등록되지 않았습니다 | RuntimeException | 400 |
| | 이메일을 입력해주세요 | RuntimeException | 400 |
| | 이미 사용 중인 이메일입니다 | RuntimeException | 409 |
| | 해당 이메일로 가입된 사용자를 찾을 수 없습니다 | RuntimeException | 404 |
| | 유효하지 않거나 만료된 인증 토큰입니다 | RuntimeException | 400 |
| | 토큰에서 정보를 추출할 수 없습니다 | RuntimeException | 400 |
| | 이메일 인증이 필요합니다 | EmailVerificationRequiredException | 403 (유지) |
| **UserSanctionService** | 유저를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| **PetService** | Pet not found, User not found | RuntimeException | 404 |
| | Pet is deleted | RuntimeException | 404 |
| | 기타 종류를 선택한 경우, 구체적인 종류를 입력해주세요 | RuntimeException | 400 |
| **UserProfileController** | 인증되지 않은 사용자입니다 | RuntimeException | 401 |
| | 현재 비밀번호와 새 비밀번호를 모두 입력해주세요 | IllegalArgumentException | 400 |
| | 인증 용도(purpose)를 지정해주세요 | IllegalArgumentException | 400 |
| **PetController** | 인증되지 않은 사용자입니다 | RuntimeException | 401 |
| **OAuth2Service** | 영구 차단/이용제한 | RuntimeException | 403 |
| | 지원하지 않는 Provider입니다 | IllegalArgumentException | 400 |
| | 회원가입 중 오류가 발생했습니다 | RuntimeException | 500 |

### 1.2 문제점
1. **일관성 부재**: "User not found" vs "유저를 찾을 수 없습니다" vs "유저 없음"
2. **예외 타입 혼용**: RuntimeException vs IllegalArgumentException (UserSanctionService)
3. **Controller try-catch**: AuthController에서 예외 잡아 수동 Map 응답 생성
4. **에러 코드 없음**: 프론트엔드에서 `errorCode`로 분기 불가

## 2. 도입할 예외 클래스

### 2.1 구조
```
domain/user/exception/
├── UserNotFoundException.java      # 404 - 유저 미존재
├── PetNotFoundException.java       # 404 - 펫 미존재
├── UserBannedException.java       # 403 - 영구 차단
├── UserSuspendedException.java    # 403 - 이용제한 (해제일 포함)
├── DuplicateUserFieldException.java # 409 - 닉네임/이메일/username 중복
├── InvalidRefreshTokenException.java # 401 - Refresh Token 관련
├── InvalidEmailVerificationTokenException.java # 400 - 인증 토큰
├── UnauthenticatedException.java  # 401 - 인증되지 않은 사용자
├── UserForbiddenException.java    # 403 - 본인 프로필만 수정 등
├── InvalidPasswordException.java  # 400 - 비밀번호 불일치
├── UserValidationException.java   # 400 - 닉네임 필수, 50자 이하 등
└── EmailVerificationRequiredException.java  # 기존 유지
```

### 2.2 ApiException 베이스
- `global/exception/ApiException.java`: `message`, `httpStatus`, `errorCode` 필드
- 도메인 예외가 이를 상속하면 GlobalExceptionHandler에서 일관 처리 가능

### 2.3 GlobalExceptionHandler 확장
- `ApiException` 상속 시 공통 핸들러 하나로 처리 가능
