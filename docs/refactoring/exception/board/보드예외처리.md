# Board 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **BoardService** | Board not found | RuntimeException | 404 |
| | User not found with userId | RuntimeException | 404 |
| | 이메일 인증이 필요합니다 | EmailVerificationRequiredException (user) | 403 |
| **CommentService** | Board not found | IllegalArgumentException | 404 |
| | Comment not found | IllegalArgumentException | 404 |
| | User not found | IllegalArgumentException | 404 |
| | Comment does not belong to the specified board | IllegalArgumentException | 400 |
| | 이메일 인증이 필요합니다 | EmailVerificationRequiredException (user) | 403 |
| **ReactionService** | Board not found | IllegalArgumentException | 404 |
| | Comment not found | IllegalArgumentException | 404 |
| | User not found | IllegalArgumentException | 404 |
| **MissingPetBoardService** | Missing pet board not found | IllegalArgumentException | 404 |
| | User not found | IllegalArgumentException | 404 |
| | 이메일 인증이 필요합니다 | EmailVerificationRequiredException (user) | 403 |
| **MissingPetCommentService** | Missing pet board not found | IllegalArgumentException | 404 |
| | User not found | IllegalArgumentException | 404 |
| | Comment not found | IllegalArgumentException | 404 |
| | Comment does not belong to the specified board | IllegalArgumentException | 400 |
| **BoardController** | userId and reactionType are required | IllegalArgumentException | 400 |
| **MissingPetBoardController** | status is required | IllegalArgumentException | 400 |
| | 유효하지 않은 상태입니다. MISSING, FOUND, RESOLVED 중... | IllegalArgumentException | 400 |
| **BoardPopularityScheduler** | catch (Exception e) | - | 스케줄러 로그만 |

### 1.2 문제점
1. **예외 타입 혼용**: BoardService는 RuntimeException, CommentService/ReactionService는 IllegalArgumentException
2. **메시지 혼재**: "Board not found" vs "Missing pet board not found" (영어)
3. **User 도메인 의존**: EmailVerificationRequiredException은 user 도메인 예외인데 board에서 사용
4. **공통 패턴**: Board/Comment/MissingPetBoard/User NotFound가 반복 → 전용 예외로 통일 필요

## 2. 도입할 예외 클래스

### 2.1 구조
```
domain/board/exception/
├── BoardNotFoundException.java         # 404 - 게시글 미존재
├── MissingPetBoardNotFoundException.java # 404 - 실종 제보 게시글 미존재
├── CommentNotFoundException.java       # 404 - 댓글 미존재
├── CommentNotBelongToBoardException.java # 400 - 댓글이 해당 게시글에 속하지 않음
└── BoardValidationException.java       # 400 - status 필수, userId/reactionType 필수, 유효하지 않은 상태
```

### 2.2 User 도메인 예외 재사용
- **UserNotFoundException**: Board 도메인에서 User 조회 실패 시 사용 (user 도메인 예외 import)
- **EmailVerificationRequiredException**: 이메일 인증 필요 시 사용 (user 도메인 예외 import)

### 2.3 ApiException 베이스
- `global/exception/ApiException.java` 상속 (기존 유지)
