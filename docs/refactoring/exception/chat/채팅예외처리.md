# Chat 도메인 예외처리 리팩토링

## 1. 현황 분석

### 1.1 예외 발생 위치 및 유형

| 서비스/컨트롤러 | 예외 메시지 | 현재 타입 | 권장 HTTP |
|----------------|-------------|-----------|-----------|
| **ChatMessageService** | 사용자를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 탈퇴한 사용자는 메시지를 보낼 수 없습니다 | IllegalStateException | 403 |
| | 채팅방을 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 채팅방 참여자가 아닙니다 | IllegalArgumentException | 403 |
| | 채팅방에 참여 중이 아닙니다 | IllegalStateException | 403 |
| | 메시지를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 본인 메시지만 삭제할 수 있습니다 | IllegalArgumentException | 403 |
| **ConversationService** | 채팅방을 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 채팅방 참여자가 아닙니다 | IllegalArgumentException | 403 |
| | 유효하지 않은 채팅방입니다 | IllegalArgumentException | 400 |
| | 사용자를 찾을 수 없습니다 | IllegalArgumentException | 404 |
| | 최소 1명의 참여자가 필요합니다 | IllegalArgumentException | 400 |
| | 최소 2명의 참여자가 필요합니다 | IllegalArgumentException | 400 |
| | 본인의 제보에는 채팅을 시작할 수 없습니다 | IllegalArgumentException | 400 |
| | 펫케어 관련 채팅방이 아닙니다 | IllegalArgumentException | 400 |
| | Participant not found | RuntimeException | 404 |
| | 이미 거래 확정을 완료했습니다 | IllegalStateException | 409 |
| | CareRequest not found | RuntimeException | 404 |
| | Provider not found | RuntimeException | 404 |
| **ChatWebSocketController** | 사용자를 찾을 수 없습니다 | RuntimeException | 404 |
| | catch (Exception) - 메시지 전송/읽음/타이핑 | - | WebSocket 에러 |

### 1.2 문제점
1. **예외 타입 혼용**: RuntimeException vs IllegalArgumentException vs IllegalStateException
2. **메시지 혼재**: "Participant not found", "CareRequest not found" (영어) vs "채팅방을 찾을 수 없습니다" (한글)
3. **User 도메인 의존**: UserNotFoundException 재사용 필요
4. **Care 도메인 의존**: confirmCareDeal에서 CareRequest, Provider 등 → Care 도메인 예외 또는 Chat 전용
5. **WebSocket 특수성**: ChatWebSocketController는 catch 후 에러 메시지 브로드캐스트 (WebSocket 응답 형식)

## 2. 도입할 예외 클래스

### 2.1 구조
```
domain/chat/exception/
├── ConversationNotFoundException.java   # 404 - 채팅방 미존재
├── ChatMessageNotFoundException.java     # 404 - 메시지 미존재
├── ConversationParticipantNotFoundException.java # 404 - 참여자 미존재
├── ChatForbiddenException.java           # 403 - 참여자 아님, 본인 메시지만 삭제, 탈퇴한 사용자 등
├── ChatValidationException.java          # 400 - 유효하지 않은 채팅방, 참여자 수, 본인 제보 등
└── ChatConflictException.java            # 409 - 이미 거래 확정 완료
```

### 2.2 User 도메인 예외 재사용
- **UserNotFoundException**: 사용자 조회 실패 시

### 2.3 Care 도메인 예외 재사용 (confirmCareDeal)
- **CareRequestNotFoundException**: CareRequest 조회 실패 시
- **UserNotFoundException**: Provider 조회 실패 시 (또는 ProviderNotFoundException 등)

### 2.4 ApiException 베이스
- `global/exception/ApiException.java` 상속 (기존 유지)

## 4. 예외 정적 메서드 제안

### ConversationNotFoundException
```java
ConversationNotFoundException // 채팅방을 찾을 수 없습니다
```

### ChatMessageNotFoundException
```java
ChatMessageNotFoundException // 메시지를 찾을 수 없습니다
```

### ConversationParticipantNotFoundException
```java
ConversationParticipantNotFoundException // 참여자 정보를 찾을 수 없습니다
```

### ChatForbiddenException
```java
ChatForbiddenException.notParticipant()      // 채팅방 참여자가 아닙니다
ChatForbiddenException.notActiveParticipant() // 채팅방에 참여 중이 아닙니다
ChatForbiddenException.ownMessageOnly()      // 본인 메시지만 삭제할 수 있습니다
ChatForbiddenException.deletedUserCannotSend() // 탈퇴한 사용자는 메시지를 보낼 수 없습니다
```

### ChatValidationException
```java
ChatValidationException.invalidConversation()   // 유효하지 않은 채팅방입니다
ChatValidationException.minParticipantsRequired(int min) // 최소 N명의 참여자가 필요합니다
ChatValidationException.ownReportCannotChat()   // 본인의 제보에는 채팅을 시작할 수 없습니다
ChatValidationException.notCareConversation()  // 펫케어 관련 채팅방이 아닙니다
```

### ChatConflictException
```java
ChatConflictException.dealAlreadyConfirmed() // 이미 거래 확정을 완료했습니다
```
