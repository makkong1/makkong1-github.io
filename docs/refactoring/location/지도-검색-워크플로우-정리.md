# Location 프론트 검색 워크플로우 정리

## 배경

- 현재 `UnifiedPetMapPage`는 지도 중심 상태와 실제 검색 기준 상태가 섞여 있다.
- 그 결과 location 탭에서 지도 이동만으로도 자동 재검색이 발생한다.
- `docs/domains/location.md`에 적어둔 `"지도는 상태를 바꾸지 않는다"` 원칙과 실제 동작이 어긋난다.

## 문제

1. 지도 중심 이동이 곧바로 API 재호출로 이어진다.
2. 사용자가 지금 보고 있는 위치와 실제 검색이 실행된 위치를 구분할 수 없다.
3. `"이 지역 검색"`처럼 명시적으로 검색을 확정하는 UX가 없다.

## 목표

- 지도 뷰 상태와 검색 상태를 분리한다.
- location 탭에서는 지도 이동만으로 재검색하지 않는다.
- 사용자가 명시적으로 검색을 확정할 수 있도록 `"이 지역 검색"` 액션을 제공한다.

## 이번 작업 범위

- `mapViewportCenter`와 `searchCenter` 분리
- location 탭 전용 `pending search area` 상태 추가
- `"이 지역 검색"` 버튼 추가
- 키워드/카테고리 검색 시 현재 지도 위치를 명시적으로 검색 기준으로 확정

## 비범위

- 지역 계층 탐색 UI 전면 도입
- 결과 리스트/바텀시트 추가
- location 외 meetup/care 탭 검색 UX 재설계

## 완료 기준

- location 탭에서 지도만 움직여서는 API가 재호출되지 않는다.
- location 탭에서 `"이 지역 검색"` 클릭 시에만 현재 지도 기준으로 결과가 갱신된다.
- 기존 키워드/카테고리 검색과 AI 추천은 깨지지 않는다.
