# 빠른 테스트 가이드

**작성일**: 2026-02-03  
**목적**: 리팩토링 사항을 빠르게 확인할 수 있는 간단한 테스트 방법

---

## 🚀 5분 테스트

### 1단계: 백엔드 확인 (1분)

**Thunder Client 또는 Postman 사용**:

```
GET http://localhost:8080/api/location-services/search?latitude=37.5665&longitude=126.9780&radius=5000
```

**확인**:
- 응답의 첫 번째 서비스에 `distance` 필드가 있는지
- `distance` 값이 숫자인지

**성공 기준**: ✅ `distance` 필드가 있고 숫자 값

---

### 2단계: 프론트엔드 확인 (2분)

**브라우저에서**:
1. 주변서비스 페이지 접속
2. 개발자 도구 열기 (F12)
3. 네트워크 탭에서 `/api/location-services/search` 요청 확인
4. 응답에서 `distance` 필드 확인

**확인**:
- API 응답에 `distance` 필드가 있는지
- 프론트엔드에서 거리 정보를 사용하는지

**성공 기준**: ✅ API 응답에 `distance` 포함, 프론트엔드에서 사용

---

### 3단계: 검색 전략 확인 (2분)

**브라우저 콘솔에서**:

```javascript
// 각 검색 시나리오 실행 후 콘솔 로그 확인
// 1. 초기 로드: "🚀 [성능 측정] 초기 로드 시작"
// 2. 위치 기반 검색: "📍 [위치 기반 검색] API 호출"
// 3. 지역 검색: "🌐 [하이브리드] 현재 데이터 범위 밖"
```

**확인**:
- 각 검색 시나리오별로 적절한 로그가 출력되는지
- 함수가 올바르게 호출되는지

**성공 기준**: ✅ 각 시나리오별로 적절한 로그 출력

---

## ✅ 통합 테스트 체크리스트

### 필수 확인 사항

- [ ] 위치 기반 검색 시 거리 정보가 표시됨
- [ ] 초기 로드 시 주변 서비스가 표시됨
- [ ] 지역 검색 시 해당 지역 서비스가 표시됨
- [ ] 검색 결과가 예상대로 나옴
- [ ] 에러 없이 동작함
