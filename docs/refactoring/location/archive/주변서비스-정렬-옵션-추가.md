# 주변서비스 정렬 옵션 추가

## 배경

- 현재 반경 검색 기본 정렬은 평점순이라 `내 주변` UX와 맞지 않는다.
- 프론트에서 받은 결과만 재정렬하면 이미 서버에서 잘린 후보 집합 안에서만 순서가 바뀌므로 정확한 거리순이 아니다.
- 사용자가 반경 필터 안에서 정렬 기준을 직접 바꿀 수 있어야 한다.

## 목표

- 반경 검색 기본 정렬을 `distance`로 전환한다.
- 반경 검색에 `distance`, `rating`, `reviews` 정렬 옵션을 추가한다.
- 프론트 location 메인 화면에서 반경 필터와 함께 정렬 옵션을 선택할 수 있게 한다.

## 정렬 정책

- `distance`: 거리 오름차순, 동순위 보정은 `rating DESC`, 마지막은 `idx ASC`
- `rating`: 평점 내림차순, 동순위 보정은 `distance ASC`, 마지막은 `idx ASC`
- `reviews`: 리뷰 수 내림차순, 동순위 보정은 `distance ASC`, 그다음 `rating DESC`, 마지막은 `idx ASC`

## 범위

- `GET /api/location-services/search` 반경 검색 분기
- location 메인 화면 정렬 UI
- location 도메인 문서 현행화

## 비범위

- 거리/평점 혼합 점수 알고리즘
- 개인화 추천 정렬
- 지역 검색 전용 정렬 UX 확장

## 검증 기준

- 반경 검색 기본 정렬이 거리순으로 바뀐다.
- 정렬 옵션 변경 시 백엔드 응답 순서가 달라진다.
- 프론트에서 반경/키워드/카테고리와 함께 정렬 옵션을 바꿔도 빌드가 깨지지 않는다.
