# Location 검색 분기·카테고리 필터 통합 리팩토링

**작성일**: 2026-03-31  
**리팩토링 목적**: `/search`와 `/recommend`에 흩어져 있던 **동일한 검색 우선순위 로직**과, 서비스 레이어에 세 번 반복되던 **카테고리 필터**를 한곳으로 모아 유지보수성을 높임

---

## 왜 했는가 (배경)

### 1. 컨트롤러에 같은 if-else가 두 벌

`LocationServiceController`에서 다음 규칙이 **`GET /search`와 `GET /recommend`에 각각 복사**되어 있었다.

- 키워드가 있으면 → 키워드 검색
- 아니면 위도·경도·반경이 모두 있으면 → 반경 검색 (`radius <= 0`이면 10km로 처리)
- 그 외 → 지역 계층 검색

**문제**: 한쪽만 수정하면 `/search`와 `/recommend`의 동작이 어긋날 수 있음. 새 엔드포인트를 추가할 때마다 같은 분기를 또 붙여야 함.

### 2. 서비스에 카테고리 필터가 세 번 중복

`searchLocationServicesByRegion`, `searchLocationServicesByLocation`, `searchLocationServicesByKeyword` 안에서  
`category1` / `category2` / `category3`를 비교하는 **동일한 스트림 필터**가 반복되었다.

**문제**: 카테고리 매칭 규칙을 바꾸려면 세 군데를 동시에 고쳐야 하고, 실수로 한 곳만 바뀌면 검색 결과가 경로마다 달라질 수 있음.

---

## 무엇을 했는가 (요약)

| 구분 | 내용 |
|------|------|
| **검색 분기 통합** | `LocationServiceService.searchLocationServices(...)` 한 메서드에서 키워드 → 반경 → 지역 순으로 위임 |
| **컨트롤러** | `/search`, `/recommend`는 위 메서드만 호출 (분기 제거) |
| **반경 기본값** | `radius > 0`이 아니면서 반경 분기에 들어온 경우 `DEFAULT_RADIUS_METERS`(10km) — 기존 컨트롤러와 동일 |
| **카테고리** | `applyCategoryFilter` → `matchesCategory` / `categoryFieldMatches`로 공통화 |

**동작(비즈니스 규칙)은 기존과 동일**하다. 구조만 정리한 리팩토링이다.

---

## 처리 흐름 (After)

```text
LocationServiceController
  GET /search   ──┐
  GET /recommend ─┼──► LocationServiceService.searchLocationServices(...)
                  │         ├─ keyword 있음     → searchLocationServicesByKeyword
                  │         ├─ lat+lng+radius  → searchLocationServicesByLocation
                  │         └─ 그 외           → searchLocationServicesByRegion
                  │
                  └─ (각 하위 메서드 내부에서 필요 시 applyCategoryFilter)
```

---

## 코드 위치

| 역할 | 클래스 / 메서드 |
|------|------------------|
| 검색 진입점 | `LocationServiceService.searchLocationServices` |
| 키워드 / 반경 / 지역별 구현 | 기존 `searchLocationServicesByKeyword` 등 (유지) |
| 카테고리 필터 | `applyCategoryFilter`, `matchesCategory`, `categoryFieldMatches` |
| HTTP | `LocationServiceController` — `searchLocationServices` 호출만 수행 |

---

## 이 리팩토링이 하지 않은 것

- **키워드 + 위치 동시 조건** (키워드 결과를 반경으로 한 번 더 자르기 등): API·제품 정책이 필요하므로 이번 범위에서 제외.
- **위도·경도만 있고 `radius` 없을 때 자동 반경**: 클라이언트 계약 변경이 될 수 있어 별도 이슈로 두는 편이 안전.
