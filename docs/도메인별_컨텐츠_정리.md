## 1️⃣ 유저 도메인 (UserDomain)

### 도메인 소개
- **역할**: 사용자 관리, 인증/인가, 반려동물 등록, 제재 시스템
- **핵심 기능**: JWT 기반 인증, 소셜 로그인, 반려동물 프로필 관리, 사용자 제재

### 주요 기능
1. **사용자 인증/인가**
   - JWT 토큰 기반 인증
   - 리프레시 토큰 관리
   - 소셜 로그인 (OAuth2)

2. **반려동물 관리**
   - 반려동물 등록/수정/삭제
   - 품종, 나이, 성별 등 상세 정보 관리

3. **프로필 관리**
   - 사용자 정보 수정
   - 위치 정보 관리

4. **사용자 제재 시스템**
   - 경고 횟수 관리
   - 일시 정지 (suspendedUntil)
   - 계정 상태 관리 (ACTIVE, SUSPENDED, BANNED)

### 기술 구현 하이라이트
- **Entity**: Users, Pet, SocialUser, UserSanction, PetVaccination
- **보안**: Spring Security + JWT
- **소프트 삭제**: isDeleted 플래그로 데이터 보존

### 성능 최적화
1. **쿼리 최적화**
   - `id`, `email`, `username`에 UNIQUE 인덱스
   - `(status, isDeleted)` 복합 인덱스로 조회 최적화

2. **캐싱**
   - 사용자 프로필 정보 캐싱
   - 권한 정보 캐싱

3. **N+1 문제 해결**
   - 소셜 계정 조회 시 Fetch Join 사용
   - 펫 목록 조회 시 배치 쿼리

### 관련 문서 링크

## 2️⃣ 게시판 도메인 (BoardDomain)

### 도메인 소개
- **역할**: 커뮤니티 게시판, 댓글, 반응 시스템, 인기글 관리
- **핵심 기능**: 게시글 CRUD, 댓글 시스템, 좋아요/싫어요, 인기글 스냅샷

### 주요 기능
1. **게시판 기능**
   - 카테고리별 게시글 작성 (자유, 정보, 질문 등)
   - 게시글 조회수 관리
   - 소프트 삭제

2. **댓글 시스템**
   - 댓글 작성/수정/삭제
   - 댓글 수 실시간 업데이트

3. **반응 시스템**
   - 좋아요/싫어요 기능
   - 반응 수 집계
   - 중복 반응 방지 (유니크 제약)

4. **인기글 스냅샷**
   - 주기적으로 인기글 스냅샷 생성
   - 조회수, 좋아요 수 기반 랭킹

5. **조회수 관리**
   - BoardViewLog로 중복 조회 방지
   - 동시성 문제 해결

### 기술 구현 하이라이트
- **Entity**: Board, Comment, BoardReaction, BoardViewLog, BoardPopularitySnapshot, CommentReaction
- **성능 최적화**: 배치 조회로 N+1 문제 해결
- **캐싱**: 게시글 조회 캐싱

### 관련 문서 링크

## 3️⃣ 펫케어 도메인 (CareDomain)

### 도메인 소개
- **역할**: 펫케어 요청/지원 시스템, 리뷰 관리
- **핵심 기능**: 요청 작성, 지원 시스템, 댓글 기반 채팅 시작, 리뷰 작성

### 주요 기능
1. **펫케어 요청**
   - 돌봄 필요 날짜, 상세 설명 작성
   - 상태 관리 (OPEN, IN_PROGRESS, COMPLETED)
   - 관련 반려동물 정보 연결

2. **지원 시스템**
   - 펫시터가 요청에 지원
   - 지원 상태 관리 (PENDING, ACCEPTED, REJECTED)
   - 보호자가 지원자 선택

3. **댓글 기반 채팅 시작**
   - 댓글 작성으로 지원 의사 표현
   - 댓글에서 "채팅하기" 버튼으로 1:1 채팅 시작

4. **리뷰 시스템**
   - 서비스 완료 후 리뷰 작성
   - 평점 및 후기 관리

### 기술 구현 하이라이트
- **Entity**: CareRequest, CareApplication, CareRequestComment, CareReview
- **상태 관리**: CareRequestStatus, CareApplicationStatus
- **채팅 연동**: 댓글에서 채팅방 자동 생성

### 관련 문서 링크
- [Care 도메인 상세 문서](https://github.com/makkong1/Petory/blob/main/docs/domains/care.md)

## 4️⃣ 실종 신고 도메인 (MissingPetDomain)

### 도메인 소개
- **역할**: 실종 동물 신고 및 관리, 목격 정보 수집
- **핵심 기능**: 실종 신고 작성, 위치 기반 검색, 상태 관리

### 주요 기능
1. **실종 신고 작성**
   - 반려동물 정보 (이름, 종, 품종, 나이, 성별, 색상)
   - 실종 날짜 및 장소
   - 위도/경도 좌표 저장
   - 사진 첨부

2. **위치 기반 검색**
   - 실종 장소 기준 주변 검색
   - 지도 연동

3. **목격 정보 수집**
   - 댓글로 목격 정보 제공
   - 실종자와 소통

4. **상태 관리**
   - MISSING (실종 중)
   - FOUND (발견됨)
   - CLOSED (종료)

### 기술 구현 하이라이트
- **Entity**: MissingPetBoard, MissingPetComment
- **위치 정보**: BigDecimal로 정확한 좌표 저장
- **알림 연동**: 실종 신고 시 알림 발송

- **평점 계산**: 평균 평점 자동 업데이트

## 7️⃣ 채팅 도메인 (ChatDomain)

### 도메인 소개
- **역할**: 실시간 채팅 시스템, 다양한 채팅방 타입 관리
- **핵심 기능**: WebSocket 기반 실시간 통신, 채팅방 타입별 관리, 메시지 읽음 처리

### 주요 기능
1. **실시간 채팅**
   - WebSocket 기반 양방향 통신
   - 실시간 메시지 전송/수신
   - 연결 상태 관리

2. **채팅방 타입**
   - **DIRECT**: 1:1 일반 채팅
   - **CARE_REQUEST**: 펫케어 요청 관련 채팅
   - **MISSING_PET**: 실종 신고 관련 채팅
   - **MEETUP**: 모임 관련 채팅

3. **메시지 관리**
   - 메시지 읽음 처리
   - 읽지 않은 메시지 수 집계
   - 메시지 히스토리 조회

4. **참여자 관리**
   - 채팅방 참여자 목록
   - 참여자 추가/제거

5. **알림 연동**
   - 새 메시지 도착 시 알림
   - 읽지 않은 메시지 알림

### 기술 구현 하이라이트
- **WebSocket**: Spring WebSocket + STOMP 프로토콜
- **Entity**: Conversation, ConversationParticipant, ChatMessage, MessageReadStatus
- **인증**: WebSocket 연결 시 JWT 토큰 검증
- **실시간성**: STOMP를 통한 Pub/Sub 메시징

### 관련 문서 링크
- [채팅 시스템 설계](https://github.com/makkong1/Petory/blob/main/docs/architecture/채팅%20시스템%20설계.md)
- [채팅 시스템 구현 차이점](https://github.com/makkong1/Petory/blob/main/docs/md/채팅_시스템_구현_차이점.md)

## 8️⃣ 신고 도메인 (ReportDomain)

### 도메인 소개
- **역할**: 사용자 신고 시스템, 관리자 처리 관리
- **핵심 기능**: 다양한 타입의 신고 처리, 관리자 조치 관리

### 주요 기능
1. **신고 시스템**
   - 게시글, 댓글, 사용자 등 다양한 타입 신고
   - 신고 사유 작성
   - 중복 신고 방지 (유니크 제약)

2. **신고 상태 관리**
   - PENDING (대기 중)
   - PROCESSED (처리 완료)
   - REJECTED (거부)

3. **관리자 조치**
   - 조치 타입 관리 (NONE, WARNING, SUSPENSION, BAN)
   - 처리자 및 처리 시간 기록
   - 관리자 메모 작성

### 기술 구현 하이라이트
- **Entity**: Report
- **신고 타입**: ReportTargetType (게시글, 댓글, 사용자 등)
- **조치 타입**: ReportActionType
- **중복 방지**: (target_type, target_idx, reporter_idx) 유니크 제약

## 9️⃣ 알림 도메인 (NotificationDomain)

### 도메인 소개
- **역할**: 사용자 알림 시스템, 다양한 알림 타입 관리
- **핵심 기능**: 실시간 알림 발송, 읽음 상태 관리

### 주요 기능
1. **알림 발송**
   - 다양한 알림 타입 지원

2. **읽음 상태 관리**
   - 읽음/안 읽음 상태 추적
   - 읽지 않은 알림 수 집계

3. **알림 타입**
   - 게시글, 댓글, 펫케어, 실종 신고, 모임 등

- **Entity**: Notification
- **알림 타입**: NotificationType
- **관련 정보**: relatedId, relatedType으로 다양한 엔티티 연결
- **읽음 추적**: isRead 플래그로 읽음 상태 관리

## 📝 각 페이지 공통 구조

각 도메인 페이지는 다음 구조로 작성:

1. **도메인 소개** (1-2문단)
   - 도메인의 역할과 목적
   - 핵심 가치

2. **주요 기능** (3-5개 항목)
   - 각 기능별 간단한 설명
   - 사용자 시나리오 (선택)

3. **기술 구현 하이라이트**
   - 주요 Entity
   - 기술적 특징
   - 성능 최적화 포인트 (해당되는 경우)

4. **관련 문서 링크**
   - GitHub 문서 링크
   - 관련 구현 가이드 링크
