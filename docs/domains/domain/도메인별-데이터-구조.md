# Petory 도메인별 데이터 구조 (백엔드 JPA 기준)

`domain/*/entity`에 정의된 **테이블·필드·열거형**을 도메인 단위로 정리했습니다.  
(면접·DB 스키마 복기·API 설계 시 “어떤 값이 어디에 붙는지” 확인용)

**관련 문서**: [도메인-구조-개요.md](./도메인-구조-개요.md) (패키지·폴더 맵)

## 정리: 스키마·관계 관점

- **관계**: 대부분 `@ManyToOne`으로 상위 엔티티(`Users`, `Board`, `CareRequest` 등)를 참조하고, 일대다는 `mappedBy` 컬렉션 또는 별도 테이블(반응·댓글)로 풉니다.
- **복합 키**: `MeetupParticipants`만 `@IdClass(MeetupParticipantsId)` 패턴을 사용합니다.
- **유니크 제약**: 게시글 반응·댓글 반응·조회 로그·신고는 “동일 사용자의 중복 행”을 막는 UK가 있습니다.
- **상태 전이**: 케어(`CareRequestStatus` / `CareApplicationStatus`), 모임(`MeetupStatus`), 에스크로(`EscrowStatus`) 등은 열거형으로 도메인 규칙을 고정합니다.

*스키마 변경 시 이 문서의 테이블명·컬럼 요약을 엔티티와 함께 갱신해 주세요.*
